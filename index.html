Prework Study Guide

HTML

  • HTML uses "elements" but I'm going to keep thinking of them as objects because that's easier for me.
  • Head element contains information that orients browser, allows you to set global variables for HTML elements
  • Body element represents visible content.
  • H tags (h1,2,3,4,5) denote headers and create larger, bold text in descending size.
  • Most of HTML works nested within element tags, beginning with the tag and ending with /tag.
  • HTML functions as the primary framework for the webpage.

CSS

  • CSS or Cascading Style Sheets apply styling to HTML elements
  • CSS can be defined internally or externally. External seems more efficient and makes it easier to implement sweeping changes.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

GIT

  • GIT is a workflow tool allowing you to manage versioning.
  • Using workflow branches in GIT allows multiple programmers to work on the same project.
  • By working on branches separately from main branch, you can prevent difficult to resolve bugs making it into main project.
  • git status: checks current branch
  • git checkout switches current branch
  • git checkout -b branch-name: creates new branch and switches to it. -b is tag for new branch
  • cd.. moves back in directory (cd stands for change directory?)
  • cd folder-name moves you into a directory within the one you're currently in
